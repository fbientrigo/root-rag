# DISParticles Stack

## Definition
The `DISParticles` stack is a `TClonesArray` of `TVectorD` objects generated by the `makeMuonDIS.py` script. It represents the products of a Pythia6 DIS interaction between a high-energy muon and a fixed target (p+ or n0).

## Data Structure
Each entry in the `DISParticles` array is a `TVectorD` with **5 elements**:

| Index | Name | Description |
|---|---|---|
| 0 | `pid` | PDG Particle ID of the produced particle. |
| 1 | `px` | Momentum X-component (GeV/c). |
| 2 | `py` | Momentum Y-component (GeV/c). |
| 3 | `pz` | Momentum Z-component (GeV/c). |
| 4 | `E` | Total Energy (GeV). |

## Origin: Pythia6 Integration
Particles are extracted from the Pythia6 event stack after a `GenerateEvent()` call:
- The script iterates through the Pythia stack (`GetN()`).
- Each particle's momentum is rotated to align with the incoming muon's direction.
- Only "final state" or relevant products (after `Pyedit(1)`) are typically kept.

## Usage in Simulation
When the `MuDISGenerator` reads an event, it loops through this stack and injects each particle into the `FairPrimaryGenerator` as a new track.

- **First Particle (Index 0)**: Used as a carrier for the DIS cross-section metadata.
- **Subsequent Particles**: Propagated as standard secondary tracks.

## Evidence Anchors
- `muonDIS/makeMuonDIS.py:250-270`: Filling the `DISParticles` array.
- `shipgen/MuDISGenerator.cxx:180-200`: Reading the `DISParticles` stack into the simulation.
