# How to use the FairShip LLMWiki

## Start points
1. Open [[HOME]].
2. Check [[WIKI_STATE]] for current status boundaries.
3. Use [[indexes/NODE_REGISTRY]] and [[indexes/MAP_REGISTRY]] for targeted navigation.

## Atomic-note workflow
- Read one atomic note at a time.
- Follow its `Connections` table to adjacent notes.
- Treat maps as relationship summaries, not proof of runtime success.

## Claim discipline
- Code anchor is not runtime validation.
- Candidate oracle observables are not final truth labels.
- Use `UNRESOLVED`/`RUNTIME_UNVALIDATED` explicitly when evidence is missing.
